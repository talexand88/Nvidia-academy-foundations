A different color per row
When every list item, table row, or data source gets its own color, no color means anything any more. The composition reads as decoration, not information. Use color only where it carries meaning — chapter identity, status, series — and let everything else stay neutral.

Gradients — Yes Or No?

No
For UI, content, charts, icons

Don't Generate Gradients Yourself

The Academy uses solid colors for backgrounds, buttons, frames, cards, text, icons, charts, and surfaces. No gradient fills on any UI element. No gradient text. No gradient buttons. No multi-stop rainbow effects.

Except
For sanctioned hero imagery

The 3D NVIDIA Hero Composition

The 3D-rendered NVIDIA "eye" composition on title cards has a built-in radial green glow. You don't generate this — grab the asset from NVDAM. The gradient lives inside the sanctioned image; it isn't a CSS or PowerPoint fill you apply.

Feedback Color Tokens

Four semantic states — used in Banners, Notifications, and any callout that conveys status. Pulled from the NVIDIA primary + secondary palette so the feedback colors stay on-brand with the rest of the system.

Info
background#B1CFE6
border#0071C5 CPU Blue
text#0071C5
Warning
background#FCEAB8
border#FAC200 Fluorite
text#5C4400 deep amber
Success
background#C0DB91
border#76B900 NVIDIA Green
text#004831 NVIDIA Secondary Green
Danger
background#E8C0D4
border#890C58 Garnet
text#890C58
Recipe. Each state pulls from a single NVIDIA palette color:
  • Background — a pale tint of the brand color, used as the fill on a Banner or callout.
  • Border — the full brand color, used as the 1 pt stroke or the left stripe.
  • Text — the brand color itself, or a darker shade pulled from the primary palette (e.g. Success uses NVIDIA Secondary Green for text). Sets up readable contrast against the pale background.
