Code uses Consolas on a white background with black text. The Academy doesn't use the colored-syntax dark theme that appears in some NVIDIA brand examples — values that need attention get a gray-200 pill highlight instead. See Code & CLI for the full spec.

Weights

Font weights control thickness for visual hierarchy and emphasis. Four steps: light, regular, semibold, bold.

Light
300 · Display, large display
Regular
400 · Body · default
Semibold
500 · Subtitles · UI labels
Bold
700 · Titles · CTAs

In Kaizen, "Semibold" is the 500 weight (not the conventional 600). In PowerPoint, the matching face is NVIDIA Sans Medium. Light (300) is reserved for very large display text where regular would feel too heavy.

Sentence Case

Do
Course overview
First word capitalized. Everything else lowercase.
Don't
Course Overview
Title case is never used in Academy materials.
Do
Enroll now
Button labels follow the same rule.
Don't
ENROLL NOW
All-caps reserved for short labels and badges only.

Acronyms (AI, GPU, VM, API), proper nouns, and product names keep their capitalization.

Emphasis

Use one weight of emphasis. Don't stack styles (no green + underline, no bold + italic). One signal is enough.

In text
The reviewer flags a slide as critical when the canvas size is wrong, the font is legacy, or a CTA fails contrast.
Color: #76B900 · Weight: Bold (700)
In code / CLI
$ kubectl get pods --namespace=academy
Highlight: --gray-200 · Foreground: black on white
Do
One important word.
Single green-bold mark per sentence.
Don't
Three green words in a row.
Loses meaning when overused. Reserve for the single most important term.

Italic & Underline

Italic and underline are available, but used sparingly. They live alongside the bold + green emphasis pattern above — pick one signal, not several.

Italic
Normal text
Italic text
For citations, foreign words, and the rare semantic emphasis that isn't already covered by bold-green.
Underline
Normal text
Underlined text
Reserved for inline hyperlinks. Avoid using underline for plain emphasis — it reads as a link.

Line Heights

Line height controls vertical space between lines. Pick by content type — looser for longer reads, tighter for headings.

1.2×  ·  Titles & display
A multi-line title sits compactly with 1.2.
Lines stay close but never touch.
1.5×  ·  Body
Body copy reads comfortably at 1.5.
This is the default for any paragraph longer than one line.
Works for slides, video, and web equally.
1.7×  ·  Long body
For documentation-style pages where three or more paragraphs stack, 1.7 keeps the rhythm relaxed.
The extra space makes long reads feel paced rather than crammed.
Use sparingly — it eats vertical space.
