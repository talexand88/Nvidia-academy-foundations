The rule

All code is set in Consolas on a dark background (#0C0C0C). Even inside a light-theme deck, code slides go dark. This matches every developer's terminal, and the syntax-highlight colors only read on dark.

CLI prompt convention

# User-level commands
$ docker --version
# Root-level commands
# systemctl restart docker
# Named host prompt
root@host:/var/log# tail -f messages
Prompt characters ($ and #) render in green to mark the line as an interactive command. Output lines below the prompt have no leading character — just the result.

Inline code in body text

Run docker run --gpus all to start a container with GPU access. The default image is nvidia/cuda:latest.

Set the environment variable CUDA_VISIBLE_DEVICES=0 to limit the container to a single GPU.

Inline code stays at body-text size in Consolas with a subtle gray fill behind it (gray-050). Not bold, not colored — just monospace and a soft background.

Don'ts

$ docker run -it nvidia/cuda
Don't
Code on a light background
Always dark, even inside a light-theme deck. The syntax colors don't read on white.
$ docker run -it
Don't
Code in NVIDIA Sans
Code is always Consolas. Sans-serif breaks alignment and developer expectation.
def train(x): return x
Don't
Custom syntax palette
Use the six sanctioned token colors. Don't import a VS Code theme.
$ docker run --gpus all --rm -it --shm-size=1g --ulimit memlock=-1 --ulimit stack=67108864 nvidia/cuda:12.3.0-base-ubuntu22.04-rocm-runtime-mainline-build-master-latest
Don't
Shrink to fit a long line
If a command wraps or shrinks below 14 pt to fit, break the example or use a shorter equivalent.
$ docker run
$ docker ps
Don't
Loose line height
Code uses 1.4× line height. Anything looser destroys the visual block.
centered code
Don't
Center-align code
Code is always left-aligned. The leading indent is part of the meaning.
