Code & CLI

Consolas. Dark background. Same recipe in slides, video, and web.

The Rule

All code is set in Consolas on a white background with black text. Values that need attention get a gray-200 pill highlight — never colored syntax tokens. NVIDIA's broader brand allows colored-on-dark code, but the Academy uses the cleaner light style throughout.
#!/bin/bash
#SBATCH -N 8 # Request 8 nodes
#SBATCH --ntasks-per-node=8 # 8 GPUs per node (total 64 MPI ranks)

CONT="nvcr.io/nvidia/hpc-benchmarks:25.10"

srun --container-image="${CONT}" \
    --container-mounts="$(pwd):/datfiles" \
    /workspace/hpl.sh --dat datfiles/HPL.dat

No Colored Syntax

The Academy doesn't use colored-on-dark syntax highlighting. Keywords, strings, functions, numbers, and comments all render as plain black text. Visual emphasis comes from the gray-pill highlight only, applied to the one or two values that matter on the line.

This is a deliberate Academy choice — the broader NVIDIA brand permits colored CLI on dark, but for course materials we hold to the simpler light style for legibility on projectors, in print, and across course platforms with varying screen calibration.

CLI Prompt Convention

# User-level commands
$ docker --version
# Root-level commands
# systemctl restart docker
# Named host prompt
root@host:/var/log# tail -f messages
Prompt characters ($ and #) stay in plain black. Output lines below the prompt have no leading character — just the result.

Inline Code In Body Text

Run docker run --gpus all to start a container with GPU access. The default image is nvidia/cuda:latest.

Set the environment variable CUDA_VISIBLE_DEVICES=0 to limit the container to a single GPU.

Inline code stays at body-text size in Consolas with a subtle gray fill behind it (gray-050). Not bold, not colored — just monospace and a soft background.

Emphasizing A Token Inside Code

When one argument, flag, or value deserves attention, wrap it in a gray-200 pill. Text stays black. One emphasis per line.

$ docker run --gpus all -it nvidia/cuda:12.3.0-base
$ kubectl get pods --namespace=academy
$ export CUDA_VISIBLE_DEVICES=0
Pill background: --gray-200 (#CCCCCC). Padding: 1px 7px. Radius: 3px. Do not use green in code — green is reserved for body-text emphasis. See Typography → Emphasis.

Don'ts

$ docker run -it nvidia/cuda
Don't
Dark background with colored syntax
The broader NVIDIA brand allows this style, but the Academy doesn't. We stay on white-with-black for legibility on projectors and in print.
$ docker run -it
Don't
Code in NVIDIA Sans
Code is always Consolas. Sans-serif breaks alignment and developer expectation.
def train(x): return x
Don't
Colored syntax tokens
All text is black. Emphasis comes from the gray-200 pill, not from token colors.
$ docker run --gpus all --rm -it --shm-size=1g --ulimit memlock=-1 --ulimit stack=67108864 nvidia/cuda:12.3.0-base-ubuntu22.04-rocm-runtime-mainline-build-master-latest
Don't
Shrink to fit a long line
If a command wraps or shrinks below 14 pt to fit, break the example or use a shorter equivalent.
$ docker run
$ docker ps
Don't
Loose line height
Code uses ~1.5× line height. Anything looser destroys the visual block.
centered code
Don't
Center-align code
Code is always left-aligned. The leading indent is part of the meaning.
$ docker run -it nvidia/cuda
Don't
Green pill highlight in CLI
Green is reserved for body-text emphasis. In code, the highlight pill is always gray-200.
