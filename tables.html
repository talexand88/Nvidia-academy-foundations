Tables

For comparing items across attributes. Green column titles, alternating row stripes, inline status marks for quick scanning.

Source. The Academy table style is set by the canonical PowerPoint table example. It's a slide-first pattern, not pulled from Kaizen (Kaizen Tables target product UI). For web / video, translate the slide spec.

Anatomy

Header Row

  • Color — column titles in NVIDIA Green, bold.
  • First cell — left blank (it's the row-heading column).
  • No border — column titles sit on white; the rest of the table provides the visual structure via row striping.

Body Rows

  • Zebra striping — alternate white / gray-050.
  • Row headings — left-aligned, medium weight.
  • Data cells — centered.
  • Empty cells — leave blank, don't write "N/A" or "—" unless the absence is meaningful.

Section Divider

  • Spans the full table — colspan all columns.
  • Single label — used as a group header inside the table body.
  • Darker gray fill — gray-100, distinct from the zebra rows.

Status Marks

  • ✓ check — green, applied to "has this", "supported", "passes".
  • ✗ x — red, applied to "doesn't have", "unsupported", "fails".
  • Blank cell — neutral / not applicable. Don't use a third symbol.

In After Effects

For motion: use the same recipe but at 28 px / 32 px / 28 px (column / row heading / body). Build the row stripes as two background rectangles, alternate, then overlay text. Animate by sliding rows in from the left (200 ms each, 50 ms stagger) rather than fading the table in as a unit.
