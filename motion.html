Motion Library

A reference library of curated Lottie animations, each paired 1:1 with a static icon from the NVDAM library. Browse, preview, grab the canonical motion for your video — the portal does not generate motion; that's what After Effects is for.

Coming in v2

The Motion library is structurally identical to the Icons library you can already use today. Icons are static-only in v1; v2 adds the matching motion pair for each icon, plus the timing standards and After Effects integration guidance shown below.

v2 — Reserved

How Motion Will Be Organized

Each icon in the NVDAM static library will have a matching Lottie animation in the curated NVIDIA workspace on lottiefiles.com. The pairing is 1:1 by design: if your slide uses the "AI compute" icon, your matching video uses its Lottie. No mixing across families, no authoring custom motion from a static SVG, no rasterizing a Lottie frame to use as a static icon.

When Motion Vs. Static

The rule of thumb: if the surface plays motion automatically, use the Lottie; otherwise use the static icon.

PowerPoint slide→ static
Course video opener→ motion
Web page (default)→ static
Web hero with anim.→ motion
Reduced-motion users→ always static
Print, screenshot, social→ static

Planned Features (v2)

What V2 Adds

  • Browsable grid of every motion icon, paired to its static counterpart
  • Inline Lottie preview — autoplay, hover-replay, scrub timeline
  • Download as .json (Lottie source) or .lottie (compressed) per icon
  • After Effects export — drop directly into a composition with timing rules pre-applied
  • Reduced-motion fallback notes per icon (which static frame to use, when)
  • Video-frame review tool (extension of the screenshot reviewer for video stills and exports)
  • After Effects template project — opening titles, section openers, "Up next" bridges, end cards
  • Motion timing reference — durations, easings, and how they relate to the audio mix (-16 LUFS, etc.)
