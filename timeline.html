Timeline & progress

Horizontal sequences of stops along a green line. Use for course progression, project phases, or chapter overviews. A separate progress bar covers in-progress percentages.

Source. Two variants from the canonical Academy PowerPoint reference — Progression (headline + description per stop) and Timeline (adds a green Timeframe eyebrow above each headline).

Number Of Stops

Sweet Spot

  • 3 stops — minimum for a meaningful sequence.
  • 4 stops — Progression default.
  • 5 stops — Timeline default; comfortably fits on a 40 × 22.5 in slide.

Edges

  • 2 stops — use a Before / After comparison block, not a timeline.
  • 6+ stops — drop descriptions, or split into two slides. Descriptions need to be legible at 22 pt; they collapse below 5 stops on a 40 × 22.5 canvas.
  • 7+ stops — switch to a vertical timeline (not documented here yet).

Progress Bar

For "X% complete" — a single horizontal fill bar. Use on course homepages, lab launch screens, and any place where the count is continuous rather than stepped.

In After Effects

Timeline animation. Build the green line as a stroke layer. Animate stops in left-to-right with each stop fading in (220 ms) followed by its description sliding up from below (180 ms, 100 ms after the dot). Stagger 300 ms per stop.

Progress bar animation. Animate the fill width from 0 → target % over 800 ms with a slight ease-out. Don't oscillate or bounce. Update the percent label on each keyframe so the number matches the bar.

Do

Course Q1 / Q2 / Q3 / Q4 4 evenly spaced stops with quarter labels. Predictable, scannable.
Do
Even spacing, regular intervals
When stops represent equal time chunks, space them evenly.
Bullet descriptions ≤ 3 items Each stop's "Description and/or main points" stays short. The slide breathes.
Do
Keep descriptions tight
If a stop needs more than 3 short bullets, it deserves its own slide.

Don't

8 stops on one slide Headlines collapse to 1 line, descriptions get cut, the line breaks the eye.
Don't
Too many stops
7+ stops belongs on two slides or in a vertical timeline.
Colored dots per status Green for done, gray for future, orange for in-progress.
Don't
Color-code the dots
All dots stay NVIDIA Green. If you need status (done / current / future), use a separate Stepper component (not documented yet) — don't bend the timeline.
Diagonal or curved line Stops arranged on a curve, with the line following.
Don't
Decorative line shapes
The line is horizontal. Curves and angles add visual noise without adding meaning.
